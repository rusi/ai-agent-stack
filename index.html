Step 1

The Basic Model (LLM)

A Large Language Model — or just "model" — is a program trained on enormous amounts of text. You send it a message, it predicts the most likely response, word by word.

No internet. No memory between conversations. No tools. Just text in, text out.

💡 Example"What is the capital of France?" → "The capital of France is Paris." The model knows this from training — it's not looking anything up.
Step 2

System Prompt & Structured Output

Models receive a system prompt — hidden instructions loaded before your message. This is the "context window."

If the system prompt says "reply in JSON," the model outputs structured data instead of prose. Now software can parse it.

⚡ This is the foundation of everything.Without structured output, tools, MCP, and agents wouldn't be possible. It's how a model "talks" to code.
Step 3

Tool Use (Function Calling)

The model outputs a structured tool request — like get_info("Paris"). A separate piece of code — the Tool Executor — runs the actual function and sends the result back.

The model then writes the final answer for the user. It never runs tools itself — it just asks for them.

🔑 Key insightThe model only outputs a JSON request. Separate code does the work. This is a crucial security boundary.
Step 4

MCP — Model Context Protocol

MCP is a universal standard for connecting AI to external services — like USB for tools.

Available tools (Gmail, Slack, GitHub…) are listed in the system prompt so the model knows what it can ask for. The MCP Server handles the actual connections.

🔌 One protocol, hundreds of integrations.Gmail, Slack, GitHub, Salesforce, Jira, PostgreSQL, Stripe, Zapier — build one MCP connector and any compatible AI can use it.
Step 5

Bash & Terminal Access

The executor can also run shell commands — install packages, read files, execute scripts. It's just another tool type.

Same pattern: model outputs bash: "npm test", executor runs it, result comes back to the model.

⚠️ Risk increases here.The AI can now execute arbitrary code on your machine. Sandboxing and permission boundaries become critical.
Step 6

AGENTS.md — The Persona

A markdown file injected into the model's context at startup. It defines who the agent is: role, rules, allowed tools, constraints.

Different AGENTS.md = different agent. Same model, totally different behavior.

📄 In practiceClaude Code uses CLAUDE.md, Cursor uses .cursorrules, Codex uses custom instructions — same concept.
Step 7

SKILL.md — Task Expertise

Skill files are loaded on-demand when the agent detects a relevant task. They contain step-by-step instructions, templates, and quality checks.

AGENTS.md = who you are (always). SKILL.md = how to do a specific task (on-demand).

📚 ExamplesPPTX creation, PDF generation, spreadsheets, frontend design, code review — each encodes best practices.
Step 8

The Agentic Loop

Instead of one request → one response, the agent keeps going. It plans, picks a tool, executes, observes the result, and decides what's next.

A single user message might trigger 15 tool calls, 30 file reads, and 5 test runs — all autonomously.

↻ Plan → Act → Observe → RepeatThis loop is what makes it an "agent" — it has agency over what happens next.
Step 9

Multi-Agent & Orchestrators

An orchestrator spawns multiple copies of this entire stack — each with different AGENTS.md, skills, and tool access.

One researches, one codes, one reviews. They pass results between each other like a virtual team.

🎯 This is the full picture.Claude Code's background QA agents, Cursor's parallel editors, and Codex's task runners all use this pattern.

Governance

5 Questions Before Approving Any AI Tool

A framework for IT, Legal, and leadership teams.

1

Where does data go?

Local, cloud API, or enterprise? Who sees prompts and outputs?

2

What can it do?

Read-only? Or can it modify files, send emails, execute code?

3

What's connected?

Which MCP tools? What permissions do they have?

4

Who controls it?

Admin oversight, audit logging, approval workflows?

5

What could go wrong?

Worst case? Can actions be reversed? Kill switch?
